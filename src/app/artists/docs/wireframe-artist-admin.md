# 어드민 아티스트 관리 페이지 - 와이어프레임

## 개요

아티스트의 CRUD 및 아티스트 간 부모-자식 관계를 관리하는 어드민 페이지.

### 핵심 도메인 개념

- **ArtistsViewEntity**: 아티스트 목록 조회 시 반환되는 엔티티
  - `artistId*`: string — 아티스트 ID
  - `artistName*`: string — 아티스트 이름
  - `artistDisplayName*`: string — 아티스트 표시 이름
  - `artistSubDisplayName*`: string | null — 아티스트 보조 표시 이름
  - `artistAbbreviatedName*`: string | null — 아티스트 약칭
  - `artistLogoImageUrl*`: string | null — 아티스트 로고 이미지 URL
  - `artistMainImageUrl*`: string | null — 아티스트 메인 이미지 URL
  - `artistDescription*`: string | null — 아티스트 설명
  - `parentArtists*`: ArtistsInArtistViewEntity[] | null — 부모 아티스트 목록
  - `childArtists*`: ArtistsInArtistViewEntity[] | null — 자식 아티스트 목록

- **ArtistsInArtistViewEntity**: 관계 내 아티스트 참조 엔티티
  - `artistId*`: string — 아티스트 ID
  - `artistName*`: string — 아티스트 이름
  - `artistDisplayName*`: string | null — 아티스트 표시 이름
  - `artistLogoImageUrl*`: string | null — 아티스트 로고 이미지 URL
  - `artistMainImageUrl*`: string | null — 아티스트 메인 이미지 URL

- **Artist Relation**: 아티스트 간 부모-자식 관계 (예: aespa → Karina)
- 한 아티스트가 **여러 부모**를 가질 수 있음 (예: Mark → NCT 127, NCT Dream)
- 관계는 다단계 가능 (그룹 → 유닛 → 하위 아티스트)

### 1.3 상태 및 인터랙션

| 요소 | 동작 |
|------|------|
| 검색 | 아티스트명으로 필터 (debounce 300ms). 하위 아티스트 매칭 시 해당 부모 카드를 자동 펼치고 매칭된 아티스트를 하이라이트 |
| 필터 드롭다운 | 전체 / 하위있음 / 하위없음 (childArtists 유무로 구분) |
| 정렬 드롭다운 | 관계순(기본) / 이름순 / 최신순 |
| 그룹 카드 | 클릭 시 접기/펼치기 (기본: 접힘) |
| ⟐ 배지 | 중복 소속 표시, 호버 시 소속 유닛 툴팁 |
| [+ 아티스트 추가] | 생성 모달 열기 |
| [수정] | 수정 모달 열기 (해당 아티스트 정보 프리필) |
| [삭제] | 확인 다이얼로그 → 삭제 |
| 뷰 전환 토글 | 유닛별(하위를 가진 아티스트만) ↔ 멤버별(리프 노드만) (다단계 그룹만 표시) |
| 페이지네이션 | 최상위 아티스트(그룹/솔로) 기준 페이징 |

### 2.2 수정 모드 차이점

| 항목 | 생성 | 수정 |
|------|------|------|
| 제목 | "아티스트 추가" | "아티스트 수정" |
| 필드 | 빈 상태 | 기존 데이터 프리필 |
| 자식 관계 | 빈 상태 | 기존 자식 표시 |
| 부모 관계 | 표시 안 함 | 읽기 전용 표시 |
| 버튼 | [취소] [생성] | [취소] [저장] |

### 2.3 관계 설정 방식

- **자식 추가**: 이 아티스트의 수정 모달에서 자식 검색/추가
  - API: `POST /v1/shuttle-operation/admin/artists/relations` — `{ parentArtistId: 이 아티스트, childArtistIds: [...] }`
- **부모 설정**: 부모가 될 아티스트의 수정 모달에서 이 아티스트를 자식으로 추가
  - 즉, 관계는 항상 **부모 → 자식** 단방향으로만 생성
- **현재 부모 표시**: 수정 모달에서 현재 부모를 읽기 전용으로 표시 (parentArtists 조회)

### 2.4 자식 아티스트 검색 컴포넌트

```
┌──────────────────────────────────────┐
│ 🔍 ae                                │
├──────────────────────────────────────┤
│  aespa          (하위 4명)            │  ← 검색 결과 드롭다운
│  Karina         (aespa)              │
│  NCT 127        (하위 3명)            │
└──────────────────────────────────────┘
```

- 검색어 입력 시 debounce로 아티스트 검색
- 선택하면 칩(chip)으로 추가됨
- [x]로 제거 가능
- 자기 자신 및 이미 선택된 아티스트는 결과에서 제외

## 6. 엣지 케이스

| 케이스 | 처리 |
|--------|------|
| 중복 소속 하위 아티스트 (Mark → 127, Dream) | ⟐ 배지로 표시, 전체 하위 아티스트 뷰에서 1회만 표시 |
| 순환 관계 방지 (A→B→A) | 부모 선택 시 자기 자신 및 하위 아티스트 선택 불가 |
| 부모 삭제 시 | 자식 아티스트는 유지, 관계만 해제 |
| 자식이 없는 그룹 | "하위 아티스트 없음" 메시지 표시 |
| 검색 결과 없음 | "검색 결과가 없습니다" 빈 상태 |
| 다단계 깊이 (3단계 이상) | 그룹 → 유닛 → 하위 아티스트까지 지원 (UI 인덴트로 구분) |
